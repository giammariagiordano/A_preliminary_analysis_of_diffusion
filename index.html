    Abstract

    The widespread adoption of software systems is clearly visible to everyone. To deal with continuous change requests and the strict time-to-market, practitioners and big companies constantly update their software systems to meet usersâ€™ requirements. This practice force developers to release immature products, neglecting best practices to reduce delivery times. As a possible result, technical debt can arise, i.e., potential design issues that can negatively impact maintenance and evolution and, in turn, increase both the time-to-market and costs. Code smells, which are sub-optimal design decisions that hinder software maintenance and increase the effort required to fix bugs, are a common symptom of technical debt. While previous research focused code smells primarily on Java programming language, the growing popularity of Python, particularly in artificial intelligence (AI)-enabled systems, has yet to be noticed. This preliminary analysis addresses this gap by exploring the diffusion of Python-specific code smells and the motivations that induce developers to introduce code smells in AI-Enabled systems. To perform our preliminary investigation, we selected 200 AI-enabled systems and 10,611 releases and used PySmell to extract information about code smells and PyDriller to extract information on commit messages. The results reveal several insights: 1) Code smells related to object-oriented principles are rarely detected in Python; 2) Complex List Comprehension is the most prevalent code smell; 3) The principal activities that can induce the formation of code smells are evolutionary.
